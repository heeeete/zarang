# ZARANG 개발 작업 요약 (Current Progress)

### 1. Supabase 인프라 구축
- **DB 스키마 설계**: `profiles`, `posts`, `post_images`, `post_likes`, `comments` 테이블 생성 및 RLS 보안 정책 수립
- **자동 프로필 생성**: DB 트리거를 사용하여 소셜 로그인 시 유저 정보를 `profiles` 테이블에 자동 동기화
- **스토리지 설정**: `post-images` 퍼블릭 버킷 생성 및 유저별 이미지 업로드 경로 구조화

### 2. 인증 및 세션 관리
- **소셜 로그인 연동**: Supabase Auth를 통한 구글 로그인 구현 (카카오, 네이버 확장 가능 구조)
- **세션 유지 최적화**: Next.js 15+ 및 `@supabase/ssr` 최신 표준인 `getAll/setAll` 쿠키 처리 로직 적용
- **로그인 리다이랙션**: `next` 쿼리 파라미터를 활용하여 로그인 후 이전 페이지로 자동 복귀하는 UX 최적화

### 3. 게시글 관리 (Full CRUD)
- **FSD 아키텍처 기반 리팩토링**: 게시글 도메인 로직(스키마, 이미지 관리 훅, API)을 `entities/post` 레이어로 응집시켜 작성/수정 기능 간 코드 중복 제거 및 의존성 정렬
- **게시글 편집 및 DnD 정렬**: `dnd-kit`을 도입하여 이미지 추가/삭제는 물론, 드래그 앤 드롭으로 자유로운 이미지 순서 변경 기능 구현
- **완전 삭제 정책 전환 (Hard Delete)**: 
    - 기존 Soft Delete 방식을 완전 삭제로 전환하여 저장 공간 최적화
    - DB `ON DELETE CASCADE` 설정 및 게시글 삭제 시 Storage 내 실제 이미지 파일 일괄 삭제 로직 구현
- **게시글 상세 조회**: 서버 사이드 정렬 및 데이터 페칭 고도화 (이미지, 댓글 순서 최적화)

### 4. 타건음(ASMR) 녹음 기능 고도화
- **고신뢰성 녹음 엔진**: `MediaRecorder`와 `AudioContext`의 `ScriptProcessor`를 결합하여 PC와 모바일 모두에서 안정적인 녹음 프로세스 구축
- **플랫폼 호환성 해결**: 브라우저별로 상이한 오디오 컨테이너(`webm`, `mp4`)를 자동 감지하고 올바른 확장자와 MIME 타입을 주입하여 모든 기기에서 끊김 없는 재생 보장
- **데이터 보존 전략**: 녹음 중지 시 오디오 파이프라인 연결을 명확히 해제(`disconnect`)하고 `File` 객체로 변환하여 업로드 무결성 확보

### 5. 이미지 처리 및 갤러리 레이아웃
- **이미지 메타데이터 자동 추출**: 업로드 시 `sharp` 라이브러리를 사용하여 이미지의 넓이/높이를 DB에 함께 저장
- **원본 비율 스마트 갤러리**: `react-photo-album` v3를 활용하여 사진의 원본 비율을 살린 세련된 갤러리 레이아웃 및 `lightbox` 전체 화면 보기 기능 구현
- **Masonry 레이아웃 도입**: `react-responsive-masonry`를 사용하여 '구경하기' 페이지에 2컬럼 가변 높이 피드 구현

### 6. 프로필 및 사용자 탐색
- **일반 사용자 프로필**: `/users/[id]` 페이지를 통해 다른 유저의 자랑거리 목록 확인 기능 추가
- **개인정보 보호**: 본인이 아닌 프로필 조회 시 이메일 등 민감 정보 노출 제한
- **프로필 수정 기능**: `/me/edit` 페이지를 통해 닉네임 변경 및 프로필 이미지 업로드 기능 구현
- **일관된 시각적 피드백**: 프로필 이미지가 없는 모든 영역에 Lucide `User` 아이콘을 기본값으로 적용

### 7. UX 및 탐색 기능 확장
- **구경하기(Explore) 페이지**: 카테고리 필터링과 무한 스크롤이 적용된 Masonry 피드 신설
- **검색 기능 고도화**: `explore_posts_with_author` 뷰(View)를 도입하여 게시글 제목/내용뿐만 아니라 **작성자 닉네임**으로도 검색이 가능하도록 검색 엔진 개선
- **헤더 컴포넌트 리팩토링**:
    - 중복된 헤더 로직을 공통 `SubHeader` 컴포넌트로 통합하여 코드 유지보수성 향상
    - 뒤로가기 버튼 클릭 시 고정된 경로가 아닌 **실제 브라우저 히스토리(`router.back`)**를 따라가도록 UX 개선
    - 메인 로고 헤더를 홈(`/`) 페이지에서만 노출하고 다른 페이지에서는 자동으로 숨기도록 조건부 렌더링 적용
- **시각적 정보 강화**: 타건음이 포함된 게시물에 볼륨 아이콘 오버레이를 적용하여 콘텐츠 특성을 직관적으로 전달
- **토스 말투(Toss Style) 적용**: 전역 안내 문구 및 피드백을 친근하고 정중한 어조(~했어요, ~할까요)로 통일
- **품질 관리**: 하이드레이션 오류 해결 및 프로젝트 전반의 TypeScript 타입 에러와 ESLint 경고 100% 정제
