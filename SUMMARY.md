# ZARANG 개발 작업 요약 (Current Progress)

### 1. Supabase 인프라 구축
- **DB 스키마 설계**: `profiles`, `posts`, `post_images`, `post_likes`, `comments` 테이블 생성 및 RLS 보안 정책 수립
- **자동 프로필 생성**: DB 트리거를 사용하여 소셜 로그인 시 유저 정보를 `profiles` 테이블에 자동 동기화
- **스토리지 설정**: `post-images` 퍼블릭 버킷 생성 및 유저별 이미지 업로드 경로 구조화

### 2. 인증 및 세션 관리
- **소셜 로그인 연동**: Supabase Auth를 통한 구글 로그인 구현 (카카오, 네이버 확장 가능 구조)
- **세션 유지 최적화**: Next.js 15+ 및 `@supabase/ssr` 최신 표준인 `getAll/setAll` 쿠키 처리 로직 적용
- **로그인 리다이랙션**: `next` 쿼리 파라미터를 활용하여 로그인 후 이전 페이지로 자동 복귀하는 UX 최적화

### 3. 게시글 관리 및 아키텍처 최적화 (Major Refactoring)
- **전역 '제목(title)' 필드 제거**: 
    - 이미지 중심의 간결한 커뮤니티를 위해 DB, API, 스키마, UI 전 영역에서 제목 필드 삭제
    - DB 뷰(`explore_posts_with_author`) 재정의 및 검색 엔진 최적화
- **데이터 페칭 API 통합**: `entities/post/api` 레이어에 `fetchPostsData`를 구축하여 서버/클라이언트 사이드 쿼리 로직을 하나로 단일화 (중복 코드 80% 제거)
- **중앙 집중형 타입 시스템**: 파편화된 `Post` 관련 인터페이스를 `entities/post`로 통합하고 `any` 타입을 완전히 제거하여 타입 안정성 확보
- **게시글 편집 및 DnD 정렬**: `dnd-kit`을 활용한 이미지 순서 변경 및 완전 삭제(Hard Delete) 정책 적용

### 4. 타건음(ASMR) 녹음 및 오디오 시스템
- **고신뢰성 녹음 엔진**: `MediaRecorder`와 `AudioContext`를 결합하여 PC/모바일 호환 녹음 프로세스 구축
- **브라우저 호환성 해결**: `webm`, `mp4` 등 상이한 컨테이너를 자동 감지하여 무결성 업로드 보장
- **시각적 아이콘 적용**: 피드 카드에 오디오 포함 여부를 알리는 사운드 아이콘 오버레이 추가

### 5. UI/UX 및 디자인 시스템 개편
- **메이슨리(Masonry) 레이아웃 전면 도입**:
    - 리스트 형태의 UI를 폐기하고 홈 피드, 구경하기, 마이페이지 등 모든 곳에 세련된 메이슨리 그리드(`PostGrid`) 적용
    - 스켈레톤 로딩 UI를 실제 메이슨리 흐름과 일치하도록 개선하여 로딩 경험 고도화
- **인스타그램 스타일 프로필**: 
    - 마이페이지 및 사용자 프로필을 좌측 아바타, 우측 통계(자랑거리, 팔로워, 팔로잉) 구조로 전면 개편
    - 이메일 노출을 제거하고 유저네임 중심의 깔끔한 헤더 구성
- **공통 헤더 리팩토링**:
    - 상세/수정 페이지용 `SubHeader` 도입 및 `router.back()` 기반의 지능적 뒤로가기 구현
    - 메인 로고 헤더를 홈(`/`) 페이지에서만 노출하도록 조건부 렌더링 적용

### 6. 소셜 상호작용 및 팔로우 시스템
- **팔로우(Follow) 기능 도입**:
    - `follows` 테이블 구축 및 RLS 정책 설정을 통한 보안 확보 (자기 자신 팔로우 금지)
    - 게시글 상세 페이지에서 작성자를 바로 팔로우할 수 있는 `ToggleFollowButton` 구현
    - 프로필 페이지에 실제 팔로워/팔로잉 카운트 실시간 연동
- **검색 기능 고도화**: 게시글 설명뿐만 아니라 **작성자 닉네임**으로도 유저를 찾을 수 있도록 검색 엔진 개선
- **무한 스크롤 최적화**: `rootMargin: 600px` 설정을 통해 사용자가 바닥에 닿기 전 다음 페이지를 미리 불러오는 부드러운 UX 구현
- **토스 말투(Toss Style) 적용**: 전역 안내 문구 및 피드백을 친근하고 정중한 어조(~했어요, ~할까요)로 통일
- **품질 관리**: 하이드레이션 오류 해결 및 ESLint/TypeScript 경고 100% 정제

### 7. 홈 피드 추천 알고리즘 고도화 (Intelligence)
- **가중치 기반 추천 엔진 (Supabase RPC)**:
    - 서버 사이드에서 동작하는 `get_home_feed` 함수를 구축하여 개인화된 피드를 실시간으로 계산
    - **가중치 체계**: 
        - **팔로우 보너스**: 팔로잉 중인 유저의 새 게시글에 높은 점수 부여
        - **내 글 일시 부스트 (UX)**: 본인이 방금 올린 글(1시간 이내)이 홈 피드 최상단에 고정되도록 최고 가중치(+200점) 부여하여 성공 피드백 강화
        - **관심 카테고리 (Personalization)**: 유저의 과거 활동을 분석하여 선호 카테고리 전체에 가중치 부여
            - *예시*: 유저가 '키보드' 글에 좋아요 5번(+40점), 댓글 2번(+20점)을 남겼다면, 해당 유저의 피드에서 모든 '키보드' 글은 기본적으로 **+60점**을 안고 상단에 노출됨
            - *효과*: 한 번 관심을 보인 분야의 '새로운 글'이 대중적 인기글보다도 상위에 위치하도록 개인화 강도를 높임
        - **상호작용 점수 (Global Popularity)**: 좋아요(1점)와 댓글(2점) 수치를 합산하여 대중적 인기 반영
        - **최신성(Freshness)**: 게시 시점으로부터 시간이 흐를수록 지수 함수 형태의 감쇠를 적용하여 최신 트렌드 유지
- **데이터 무결성 및 보안 강화**:
    - **타입 안정성**: SQL의 `numeric` 결과를 `FLOAT8`로 명시적 캐스팅하여 엄격한 타입 매칭 에러 해결
    - **보안 권고 준수**: 유저 정보를 가져올 때 쿠키 기반의 `getSession` 대신 서버 인증을 거치는 `getUser()`를 사용하여 보안성 확보
- **FSD 아키텍처 기반 분리**:
    - **Entity (Data)**: 추천 게시글 전용 `fetchHomePosts` API 구축
    - **Widget (Logic)**: 무한 스크롤과 추천 로직을 결합한 `useHomePosts` 커스텀 훅 설계
    - **Page (Render)**: 서버 컴포넌트(`HomePage.tsx`)에서 첫 12개의 추천 게시글을 즉시 페칭하여 LCP(Largest Contentful Paint) 최적화
