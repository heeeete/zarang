# ZARANG (자랑)

> 키보드, 마우스, 데스크 셋업 등 자신의 취향이 담긴 아이템을 사진과 소리로 자랑하는 커뮤니티

**인원**: 개인 | **기간**: 26.03.17 ~ 26.03.28 | **[사이트](https://zarang.vercel.app)**

---

## ✨ 핵심 기능

- 개인화 홈 피드 추천 (가중치 기반 스코어링, Supabase RPC)
- 고해상도 이미지 처리 및 업로드 (Canvas API, 클라이언트 직접 업로드)
- 실시간 알림 및 채팅 (DB 트리거, Realtime)
- 타건음 녹음 (MediaRecorder, Web Audio API)

---

## 🧠 기술적 문제 해결

### 1. 고해상도 이미지 다중 선택 시 UI 버벅임 해결

- **문제**: 3MB가 넘는 고해상도 사진을 여러 장 선택하면 미리보기가 뜨기까지 한참 걸리고, 그동안 버튼과 스크롤이 굳어버리는 현상이 있었습니다.
- **원인**: 원본 이미지(Blob)를 그대로 `<img>` 태그에 넣으면 브라우저가 디코딩·렌더링하는 과정에서 메인 스레드를 과도하게 사용하게 됩니다.
- **해결**: 화면에 보여주는 데이터와 서버로 보낼 데이터를 분리했습니다. `Canvas API`로 300px 썸네일을 즉시 생성해 미리보기에 사용하고, 실제 업로드는 원본 `File` 객체를 백그라운드에서 처리했습니다.
- **결과**: 사진을 아무리 많이 선택해도 0.1초 만에 미리보기가 뜨고 UI가 멈추지 않습니다.

### 2. Vercel 업로드 용량 제한(4.5MB) 해결

- **문제**: Vercel 서버리스 함수의 요청 용량 제한으로 고해상도 원본 사진을 서버로 전송할 수 없었습니다.
- **해결**: 서버를 거치지 않고 브라우저에서 이미지를 가공한 뒤 Supabase Storage로 직접 업로드했습니다. `createImageBitmap`으로 회전 정보를 보정하고 품질 0.8로 압축해 용량을 10% 이상 줄였습니다.

### 3. 피드 목록 조회 속도 개선

- **문제**: 좋아요·댓글 수와 추천 점수를 계산해 피드를 보여줄 때 데이터가 많아질수록 조회가 느려졌습니다.
- **해결**: 연산 부하가 큰 로직을 Postgres 함수(`get_home_feed`)로 구현하고 RPC로 호출했습니다. 매번 집계하지 않도록 카운트 컬럼을 추가하고 트리거로 자동 갱신하는 역정규화 방식을 적용했습니다.

### 4. DB 트리거를 활용한 자동 알림 시스템

- **문제**: 댓글·답글 생성 시 알림을 안정적으로 보내야 했습니다.
- **해결**: 애플리케이션 레이어가 아닌 DB 트리거로 자동화했습니다. `comments` 테이블에 데이터가 들어올 때마다 `handle_new_comment_notification` 함수가 실행되어 알림 대상을 찾고 `notifications` 테이블에 자동으로 기록합니다.
- **결과**: 어떤 경로로 댓글이 생성되더라도 알림이 누락되지 않는 구조를 만들었습니다.

### 5. 채팅 및 검색 최적화

- **채팅**: 안 읽은 메시지 수를 계산하기 위해 수천 개의 메시지를 클라이언트로 가져오던 비효율을 Postgres 함수(`get_unread_counts`) + RPC 호출로 개선했습니다.
- **검색**: 입력마다 API 요청이 발생하는 문제를 500ms 디바운스로 해결했습니다.
